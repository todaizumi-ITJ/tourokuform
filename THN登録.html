THN 登録フォーム

📘 使い方ガイド

このフォームは何をするものですか？

著作権侵害の連絡が来た人を「仮登録（THN）」するフォームです。
書類やメールをもとに情報を入力し、JSONファイルに保存します。
保存したJSONは別のClaudeに渡して通知判定などの処理に使います。

📋 基本の流れ（3ステップ）

STEP 1
情報取得元を選ぶ（プロバイダ / 弁護士 / 本人 / その他）
書類がプロバイダからなら「プロバイダ」をクリック。
STEP 2
フォームに入力する
氏名・住所は必須です。書類に書いてある通りに入力してください。
IPアドレス・タイムスタンプ・IPNも書類にあれば入力します。
STEP 3
「リストに追加」→「JSONエクスポート」
複数件まとめて登録してからエクスポートできます。
保存されたJSONファイルをClaudeに渡してください。

📄 各フィールドの入力例

氏名 山田 太郎　（書類の名前をそのまま）
氏名カナ ヤマダ タロウ　（あれば）
住所 東京都新宿区西新宿1-1-1
IPアドレス 60.122.20.131　（書類記載のIP）
タイムスタンプ 2024/02/22 19:09:49　（書類の日時）
権利者 株式会社Plum　（著作権者名）
IPN 095FFF　（書類に記載の識別コード）

⚠️ よくある間違い

  • 氏名・住所は必ず入力してください（空欄だと登録できません）
  • 本人申告の場合は「本人」タブを選ぶと、IP・IPN欄はグレーになり入力不要になります
  • エクスポートしたJSONはそのままClaudeに貼り付けて渡せます
  • Claudeに入力させる場合は下の「Claudeに頼む方法」を読んでください

💬 Claudeに入力を頼む場合

書類の内容をClaudeに見せて、以下のように頼むだけでOKです：

「THN登録.htmlのJSON仕様に従って、以下の情報をJSON形式に変換して。
氏名：山田太郎、住所：東京都…、IP：60.122.20.131、
タイムスタンプ：2024/02/22 19:09:49、権利者：株式会社Plum、
IPN：095FFF、情報取得元：プロバイダ（ソフトバンク）」

→ Claudeが出力したJSONをコピーして、このページの「📥 JSONインポート」から取り込めます。

THN（仮登録番号）の設計概要

THN（Temporary Human Number）とは、IPN確定前の段階で人物を仮登録するためのIDです。
プロバイダ開示・弁護士提供・本人申告などの経路で届いた情報を統一フォーマットで保存し、
後続の通知判定（app.py）・HN昇格・MHN統合へ渡します。

▍ アーキテクチャ上の位置づけ

書類・メール・本人申告
　　　↓
THN（仮登録）　← このフォームで作成
　　　↓ IPN確定・同一人物判定
HN（案件番号）　← human_all.jsonに登録
　　　↓ 複数HNが同一人物と判明
MHN（人物マスターID）　← 束ねID

▍ source_type 別の必須フィールドと処理方針

source_type 必須フィールド IPN 後処理
provider 氏名・住所・IP・timestamp・provider・rights_holder・ipn_id ✅ 必須 即時通知判定可
lawyer 氏名・住所・IP・timestamp・provider・rights_holder・ipn_id ✅ 必須 即時通知判定可
self 氏名・住所のみ ❌ なし プロバイダTHNとのマージ待ち
other 氏名・住所（最低限） 任意 都度判断

▍ thn_id の生成規則

// ブラウザ側（このフォーム）
thn_id = "THN-" + Date.now();
// 例: "THN-1741234567890"

// Claudeが生成する場合
// 省略可（フォームがインポート時に自動補完）
// または "THN-" + 任意のユニーク値 を明示してもよい

▍ thn_status の遷移

pending　→（IPN確定・判定完了）→　promoted（HN昇格済み）
　　　　→（マージ済み）→　merged（別HNに統合）
　　　　→（判定不要と確認）→　dismissed（破棄）

▍ 通知判定（app.py）との接続方法

エクスポートされたJSONをapp.pyに渡すには、現在は手動でClaudeに渡してください。
将来的には POST /api/thn/batch エンドポイントへ直接送信する設計を予定。
app.pyの照合ロジックは 氏名正規化 → 住所前20文字比較 → IPN → CP解決 → 包括和解判定 の順で処理されます。

⚠️ 既知の制約・注意点

  • thn_id は Date.now() ベースのため、同一ミリ秒に2件登録すると重複する可能性あり（実運用では無視できる程度）
  • インポートJSON内に thn_id が存在する場合はそのまま使用される（上書きなし）
  • selfの場合、後からプロバイダTHNが来た際のマージロジックはapp.py側で実装が必要
  • このフォームはサーバーなし・ローカルHTMLのため、登録データはページリロードで消える。エクスポートを忘れずに。
