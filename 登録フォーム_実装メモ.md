# 登録フォーム（登録.html）実装メモ

更新日: 2026-03-06

---

## 概要

著作権侵害案件の登録データを作成するためのHTML単体アプリ。サーバー不要で、ブラウザだけで動作する。3種類の入力ソースに対応し、JSON形式で出力する。

---

## 3つのタブ

| タブ | source_type | 入力元 | 主な用途 |
|------|------------|--------|----------|
| プロバイダ | `provider` | プロバイダ開示通知 | app.pyで通知判定 |
| 弁護士 | `lawyer` | 弁護士事務所の受任通知 | 弁護士案件DB登録 |
| 本人申告 | `self` | 本人（契約者）からの申告 | IPN判明後に通知判定 |

---

## 主な機能

### 入力方法
- **手入力フォーム**: 折りたたみ式（デフォルト閉）。1件ずつ手入力
- **Excelインポート**: HTML/Excel/CSVファイルをドラッグ＆ドロップまたはファイル選択。シートプレビュー後に一括取り込み
- **JSONインポート**: JSON配列をテキストエリアに貼り付けて取り込み

### 自動修正（autoFix）
リスト追加時に自動で実行。Excelインポート時も全件に自動適用。

| 修正項目 | 内容 |
|---------|------|
| rights_holder 人名判定 | 人名（漢字2-10文字、法人キーワードなし）→ 空に変更 |
| timestamp 正規化 | 5種類のパターン → `YYYY/MM/DD HH:mm:ss` に統一 |
| ip_address スペース除去 | 半角スペース・全角スペースを除去 |
| ipn_id 数字のみ | 数字だけのipn_idは不正 → 空に変更 |
| contract_name 補完 | 空の場合、addressから法人名を抽出して補完 |

### 修正レポート
- カード型UIで結果を表示（取り込み数、問題なし、自動修正、要確認、スキップ数）
- 修正内訳も詳細表示
- Claude確認が必要な件がある場合はオレンジ色の警告表示

### Claude連携
- 「修正プロンプトをコピー」ボタンでClaudeに渡すプロンプト+JSONをコピー
- Claudeが修正したJSONは「JSONインポート」で取り込み直し可能

### 出力
- **JSONダウンロード**: `provider_YYYY-MM-DD.json` / `lawyer_YYYY-MM-DD.json` / `self_YYYY-MM-DD.json`
- **JSONコピー**: クリップボードにコピー（file://プロトコル対応済み）

## マスターデータ（JS定数として埋め込み）

| 定数 | 件数 | 元ファイル |
|------|------|-----------|
| `PROVIDER_MASTER` | 514件 | master_provider_group_20260306.json |
| `AGENT_MASTER` | 839件 | master_agent_20260306.json |

### プロバイダコード入力
- テキストボックスにコード（P1, P2等）を入力 → 横にプロバイダ名を確認表示
- ドロップダウンからも選択可能（双方向同期）
- `PROVIDER_MAP` で code → name 変換

### 弁護士コード入力
- テキストボックスにコード（L1, L2等）を入力 → 横に弁護士名+事務所名を確認表示
- コードが未発行の場合は注意書き表示
- `AGENT_MAP` で code → {name, office} 変換

## 技術仕様

- **フレームワーク**: なし（Vanilla JS）
- **外部ライブラリ**: SheetJS（xlsx.full.min.js） — CDNから動的読み込み
- **状態管理**: `providerList`, `lawyerList`, `selfList` の3つのJS配列
- **重複チェック**: `normName()` + `normAddr()`（先頭12文字）で照合
- **クリップボード**: `execCommand('copy')` → `navigator.clipboard` → 手動コピーモーダル（3段フォールバック）
- **file://プロトコル**: 対応済み（navigator.clipboard制限を回避）

## XLS_MAP（Excel列名マッピング）

```
contract_name, user_name → contract_name
contract_name_kana, user_name_kana → contract_name_kana
address, postal_code, phone, email → そのまま
ip_address, port, timestamp → そのまま
provider, rights_holder, rights_holder_new → rights_holder
ipn_id, lawyer_id → そのまま
google_drive_url, memo → そのまま
pdf_filename → memo に追記
```

---

## 既知の注意点

- HTMLファイルはローカルで開く前提（file://プロトコル）
- SheetJS読み込みにはインターネット接続が必要（初回のみ）
- PROVIDER_MASTERとAGENT_MASTERはHTML内にハードコードされているため、マスター更新時はHTML再生成が必要
- Excelインポート後は `_resetExcelCard()` で状態をクリアし、連続インポートに対応
