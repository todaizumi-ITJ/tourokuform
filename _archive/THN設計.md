# THN（仮登録番号）設計書

作成日: 2026-03-05

---

## 1. THNとは

人の情報を片っ端から受け取るための仮登録エントリ。
IPNが確定した時点でHNに昇格する。

```
THN（仮登録）→ IPN確定 → HN（正式登録）→ 通知判定
```

---

## 2. 情報取得元の種類と特性

| 情報取得元 | 内容 | 付随情報 |
|-----------|------|---------|
| **プロバイダ** | 発信者情報開示請求の結果 | 全フィールド揃う |
| **弁護士** | 弁護士経由の通知 | 全フィールド揃う |
| **本人** | 本人申告 | 氏名・住所のみ。後からプロバイダ経由で来るので放置でOK |
| **その他** | ケースバイケース | 要確認 |

プロバイダ・弁護士経由は実質すべての情報が揃うため、受け取った時点でHN昇格の条件がほぼ揃う。

---

## 3. THNのフィールド定義

### 必須（全取得元共通）

| フィールド | 説明 |
|-----------|------|
| thn_id | THN番号（自動採番） |
| 氏名 | contract_name |
| 住所 | address |
| 情報取得元 | source_type（provider / lawyer / self / other） |

### 自動付与（プロバイダ・弁護士経由）

| フィールド | 説明 |
|-----------|------|
| IP | IPアドレス |
| タイムスタンプ | 検出日時 |
| プロバイダ名 | provider |
| 権利者 | rights_holder |
| IPN | ipn_serial（あれば） |

### 任意

| フィールド | 説明 |
|-----------|------|
| カナ | contract_name_kana |
| 郵便番号 | postal_code |
| 電話 | phone |
| メール | email |

## 5. THN → HN 昇格フロー

```
THN登録
    │
    ├─ IPNあり → HN昇格（即時）
    │
    └─ IPNなし
           │
           ├─ IP + タイムスタンプあり
           │      → db_ip_lookupでIPN後引き
           │          ├─ IPN発見 → HN昇格
           │          └─ IPN未発見 → 昇格待ち（週次レポート対象）
           │
           └─ IP + タイムスタンプなし（本人申告のみ）
                  → 昇格待ち
                  → 後からプロバイダTHNが来たら氏名・住所照合でマージ → HN昇格
```

---

## 6. 本人申告THNの扱い

本人からの登録は氏名・住所のみで、IPN・IP・タイムスタンプがない。
ただし、後からプロバイダ経由で同一人物のTHNが届くため以下の流れになる。

```
本人THN登録（氏名・住所のみ）
    ↓ 放置（昇格待ち）
    ↓
後日プロバイダTHN登録
    ↓
氏名・住所照合で同一人物と判定
    ↓
マージ → IP・タイムスタンプ・IPN付与 → HN昇格
```

本人THNは「後から来るプロバイダTHNとのマージ待ちエントリ」として機能する。

---

## 7. THNのステータス管理

| ステータス | 意味 |
|-----------|------|
| `pending` | 登録済み・昇格待ち |
| `merged` | 別THNにマージ済み |
| `upgraded` | HNに昇格済み |
| `review` | 週次レポート対象（人の判断が必要） |

---

## 8. 週次AIレポート

HNになれていないTHNを週1回スコアリングし、担当者に提示する。

### スコアリング基準（案）

| 条件 | スコア |
|------|--------|
| 情報取得元 = プロバイダ | +3 |
| 情報取得元 = 弁護士 | +2 |
| IP + タイムスタンプあり | +2 |
| 同姓同名HNと住所前20文字一致 | +3 |
| 同姓同名HNと住所前12文字一致 | +2 |
| 権利者情報あり | +1 |

| スコア | 判定 | アクション |
|--------|------|-----------|
| 7以上 | HN昇格を強く推奨 | 担当者が確認・手動昇格 |
| 4〜6 | 要確認 | 目視確認推奨 |
| 3以下 | 情報不足 | 保留・追加情報待ち |

---

## 9. THNが同姓同名判定に果たす役割

HNになれないTHNも、同姓同名判定の補助データとして活用する。

```
新規案件の氏名照合
    ↓
HNと照合（既存）
    +
THNとも照合（補助）
    ↓
THNにマッチ → 同一人物候補として判定精度を上げる
```

---

## 10. HNとの関係まとめ

```
THN：仮登録。人を片っ端から入れる箱
HN ：正式登録。IPNが確定した案件のみ
MHN：同一人物を束ねる親ID（HN昇格後に付与）

THN → （IPN確定）→ HN → （同一人物判定）→ MHN
```
